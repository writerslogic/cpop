## Adding Your Organization or Project

We'd love to hear how you're using witnessd_jitter! To be added to this list:

### Requirements

- Your organization or project must use witnessd_jitter in a production environment
- You must be authorized to publicly share this information
- Provide accurate and up-to-date information

### How to Submit

1. **Fork this repository**

2. **Edit this file** and add your entry to the appropriate table:

   For organizations:
   ```markdown
   | [Your Organization](https://your-website.com) | Brief description of use case | Year |
   ```

   For open source projects:
   ```markdown
   | [Project Name](https://github.com/...) | Brief description | [GitHub](https://github.com/...) |
   ```

3. **Submit a pull request** with:
   - Title: `Add [Organization/Project Name] to adopters`
   - Description: Brief explanation of how you use witnessd_jitter

### Information Guidelines

- **Organization/Project name**: Include a link to your website or repository
- **Description**: Keep it concise (1-2 sentences) describing your use case
- **Since**: Year you started using witnessd_jitter in production (organizations only)
- **Link**: For projects, include a link to the source code

### Questions?

If you have questions about being listed or want to share your story in more detail, feel free to:
- Open a [discussion](https://github.com/writerslogic/witnessd/discussions)
- Contact us at adopters@writerslogic.com
