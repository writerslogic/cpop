# Security Policy

This document outlines the security policy for the `cpop_protocol` crate.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please report it responsibly.

**:warning: Please do NOT report security vulnerabilities through public GitHub issues.**

Report vulnerabilities via:

1. **GitHub Security Advisories**: Navigate to the Security tab of the repository and click "Report a vulnerability".
2. **Email**: Send details to `security@writerslogic.com`.

## Security Commitment

- We will acknowledge receipt of your report within 48 hours.
- We will work with you to understand and validate the issue.
- We commit to a coordinated disclosure process.

## Safe Rust

This crate favors safe Rust code. All use of `unsafe` must be clearly documented and justified with a security rationale.
